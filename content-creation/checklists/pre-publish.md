# Pre-Publish Checklist

Run this before ANY content goes live.

---

## Voice Check

- [ ] **Sounds like them?** Read aloud. Does it match their voice?
- [ ] **Uses their words?** Check against "words to use" in voice.md
- [ ] **Avoids banned words?** Check against "words to avoid"
- [ ] **Right dimensions?** Casual/formal, serious/funny match the target?
- [ ] **Could competitor use this?** If yes, make it more specific

---

## Anti-Generic Check

- [ ] **Specific > vague?** Replace any generic claims with specifics
- [ ] **Hook in first line?** Would this stop scrolling?
- [ ] **Story or proof?** Is there something concrete, not just claims?
- [ ] **Surprising element?** Anything unexpected?
- [ ] **Would this survive the stakes scenario?**

---

## Customer Check

- [ ] **Speaking to their pain?** Reference from customers.md
- [ ] **Using their language?** Actual phrases they use
- [ ] **Clear transformation?** What do they become after this?

---

## Format Check

- [ ] **Matches platform?** Length, tone, format right for channel
- [ ] **CTA clear?** One obvious action
- [ ] **Scannable?** Subheads, short paragraphs, white space
- [ ] **Mobile-friendly?** Check on phone

---

## Technical Check

- [ ] **Spelling/grammar?** Run check
- [ ] **Links work?** Click every link
- [ ] **Images load?** Check all media
- [ ] **Meta/SEO?** Title, description, tags (if applicable)

---

## Final Questions

1. **Would the founder send this?** If not, why not?
2. **Would a customer share this?** What's share-worthy?
3. **Does this build trust?** Or erode it?

---

## If Anything Fails

- **Voice fail:** Rewrite with story.md quotes in mind
- **Generic fail:** Run back through creativity engine
- **Customer fail:** Re-read customers.md, try again
- **Technical fail:** Fix before publishing

---

## Post-Publish

- [ ] Scheduled update to learnings.md
- [ ] Monitoring plan for engagement
- [ ] Follow-up content identified
