# Corrections - March 8, 2026

## Agent Messaging Bug: UUID vs Name Addressing
**Context:** Sam Cloud couldn't receive A2A messages I sent
**Learning:** `agent_messages` table has inconsistent addressing - some use names (`saber`, `sybil`), some use UUIDs (`415a84a4-af9e-4c98-9d48-040834436e44`). Inbox scripts filter by name → UUID-addressed messages get missed. **Always use agent names (not UUIDs) in `from_agent` and `to_agent` fields.**
**Stakes:** high (breaks inter-agent communication entirely)

## SUPABASE_URL Typo in Cloud Deployment Docs
**Context:** Scout deployment failed, discovered critical typo in setup documentation
**Learning:** Documentation had `.supabase.com` instead of `.supabase.co` - wrong TLD breaks all database connections. This typo was in v9/v10 of cloud setup guide and could have broken multiple deployments.
**Stakes:** high (breaks entire agent functionality)

## Gateway Port Conflict During Cloud Deployment
**Context:** Scout changed gateway.port to 8080 during deployment, broke system
**Learning:** gateway.port should stay 18789 (default), not 8080. Port conflicts cause OpenClaw gateway failures. When agents edit config, they may inadvertently break networking.
**Stakes:** high (breaks agent entirely)
