# Corrections - March 6, 2026

## Santos Cloud API Auth Middleware Block
**Context:** Santos Cloud worked in gateway UI but returned "Auth required" for /v1/chat/completions after redeploy
**Learning:** New wrapper version has `requireDashboardAuth` middleware that blocks ALL routes including /v1. Fixed by adding `/v1/` exemption to middleware (lines 1336-1337 in server.js). Wrapper auth (Basic) and gateway auth (Bearer) are separate layers.
**Stakes:** high (breaks all API access to cloud agents)

## Webchat Agent Name Display Bug
**Context:** All agents were showing as "Sam" in the webchat platform regardless of actual agent
**Learning:** Root cause was hardcoded `subAgents` array in Chat.tsx lines 418-425. Fixed by adding `agentName` prop to component. Always check for hardcoded values when dynamic behavior expected.
**Stakes:** medium (user confusion, wrong agent attribution)
