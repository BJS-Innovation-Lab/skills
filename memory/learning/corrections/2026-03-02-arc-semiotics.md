## The Correction

**What I was doing:**
- BFS exploration of game states
- Systematic testing of all actions
- Looking for patterns in code/pixels
- Spending 30+ minutes to understand mechanics

**What actually works:**
- Read the visual language FIRST
- Apply human semiotics (plus = action, matching = goal)
- Form hypothesis from visual cues
- Test with minimal actions

---

## The Principle: Matching = Meaning

| Match Type | Human Interpretation |
|------------|---------------------|
| Same shape | "Combine them" / Goal state |
| Same color | Same system / Related function |
| Symmetry | Intentional design |

**Example:** Yellow pickup + yellow life bar = "more moves" (no testing needed)

---

## The 30-Second Rule

> If a human solves Level 1 in 29 moves, understand the mechanic in 30 seconds of reasoning — not 30 minutes of BFS.

---

## Application

Before exploring ANY game:
1. What symbols are on screen?
2. What do they mean? (universal UI semiotics)
3. What colors/shapes repeat? (matching = relationships)
4. Form ONE hypothesis from visual cues
5. Test with minimal actions
