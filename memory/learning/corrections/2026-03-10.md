## Agent Webhook Registration Requirements

**Context:** Scout wasn't receiving webhooks properly in Frontier Lab setup.

**Learning:** Agent webhook registration requires TWO steps, not one:
1. Add agent UUID to `known_agents` table with webhook_url
2. Agent needs `senderType: "agent"` in POST body for auth

**Stakes:** medium - Prevents wasted debugging time on multi-agent webhook systems

## Webhook Payload vs A2A Fallback

**Context:** Webhooks were working but message content was empty.

**Learning:** Direct webhook payloads include full message content. A2A fallback patterns may strip content fields. Always verify payload structure when debugging agent communication.

**Stakes:** medium - Critical for agent-to-agent communication debugging

## Agent Script Distribution Pattern

**Context:** New agents needed frontier-respond.cjs script to participate in Frontier Lab.

**Learning:** Can distribute scripts to agents via webhooks instead of manual setup. Send script content + implementation instructions through the webhook system itself.

**Stakes:** low - Useful pattern for agent onboarding but not critical
