# Corrections - March 10, 2026

## Frontier Lab Agent Registration Missing UUIDs
**Context:** Working on Frontier Lab multi-agent collaboration, Scout was receiving webhooks but couldn't respond.
**Learning:** New agents need their UUID added to the `known_agents` table to be recognized by the webhook system. The agent UUID must match exactly what's in their configuration, and the webhook_url field must point to their gateway.
**Stakes:** medium - blocks agent participation in multi-platform collaborations

## Webhook Payload Using A2A Fallback Instead of Direct Content
**Context:** Debugging why webhook payloads were empty or missing actual message content.
**Learning:** The webhook system was falling back to A2A message format instead of using the direct webhook payload content. This happens when the webhook routing isn't properly configured for the specific platform integration.
**Stakes:** medium - prevents agents from receiving actual user messages

## Frontier Lab POST Authentication Requires senderType Field
**Context:** Scout could receive webhooks but auth errors when trying to POST responses back to the platform.
**Learning:** Frontier Lab's respond_url endpoint requires `senderType: "agent"` in the POST body for authentication, not just the content and sender_name. Missing this field causes 401 errors even with valid content.
**Stakes:** low - specific to Frontier Lab integration but easy fix once known
