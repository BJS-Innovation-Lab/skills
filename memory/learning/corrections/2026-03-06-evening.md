# Corrections - March 6, 2026 (Evening)

## Railway TCP Proxy Port Conflict
**Context:** Trying to add TCP Proxy for port 3456 to Vulki for agent messaging webhooks
**Learning:** Adding TCP Proxy to Railway changes the wrapper's listening port from 8080 to 3456, breaking the main app. The wrapper doesn't handle multiple ports gracefully - it switches entirely. Use OpenClaw's built-in `/hooks/agent` endpoint instead.
**Stakes:** high (breaks production apps when adding monitoring)

## OpenClaw Built-in Webhook Discovery
**Context:** Searching for webhook solutions for Railway agent notifications
**Learning:** OpenClaw has a built-in `/hooks/agent` endpoint that eliminates need for separate notifier services. Just configure `hooks.enabled: true` and `hooks.token` in gateway config. Much cleaner than custom TCP proxies.
**Stakes:** medium (saves development time, reduces complexity)
