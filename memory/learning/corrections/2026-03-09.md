# Corrections - March 9, 2026

## Frontier Lab Security Fence Blocking Webhook Responses
**Context:** Working on Frontier Lab agent responses via webhooks. Agents were receiving messages but couldn't respond back to the platform.
**Learning:** OpenClaw's security fence blocks curl commands even from hook sessions, preventing agents from posting to respond_url callbacks. Solution: Parse respond_url inside the fence, validate against trusted domain pattern, then POST directly.
**Stakes:** high - critical for multi-platform agent integration

## Scout Brain Symlinks Pointing to Wrong Agent
**Context:** Deploying Scout agent to Railway, found she was responding with Vulkimi's personality.
**Learning:** Brain symlinks in deployment containers can get misconfigured during setup. Always verify that IDENTITY.md, SOUL.md, etc. point to the correct agent's folder, not another agent's.
**Stakes:** medium - wrong personality confuses users and breaks agent identity

## Config Files Override Environment Variables
**Context:** Updated Scout's Anthropic API key in Railway environment variables but agent still used old key.
**Learning:** OpenClaw's auth-profiles.json file takes precedence over environment variables. Must update both the env var AND the config file when changing credentials.
**Stakes:** medium - causes authentication failures and service interruptions
