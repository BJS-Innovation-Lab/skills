# Surprises — 2026-03-04

## Etsy API Requires Manual Approval
**Context:** Setting up Etsy API for StorybookRetro automation.
**Learning:** Etsy API applications require manual review and approval before keys become active - not instant like most APIs. This blocked the automation work despite having valid API credentials. Need to plan for approval delays in API-dependent projects.
**Stakes:** medium (project timeline delays, need backup plans for API approvals)

## Build Failures Cascade After Reverts
**Context:** Webchat platform deployment issues after webhook routing fix.
**Learning:** Fixed webhook routing bug, but merge caused Vercel build failures. Even after reverting the changes, builds continued failing. Infrastructure issues can persist beyond code changes - reverting doesn't always restore stability.
**Stakes:** medium (deployment instability, hard to debug without dashboard access)

## A2A Daemon Crashes from Missing Config
**Context:** Vulki/Vulkimi daemons not receiving A2A messages.
**Learning:** A2A daemons crash silently on startup when `A2A_AGENT_ID` environment variable is not set. No error logs, just silent failure. Added `agent.conf.example` to template to prevent this. Config validation should happen early with clear error messages.
**Stakes:** medium (silent failures are hard to debug, affects team communication)

## Learning Extraction Script Missing 97% of Learnings
**Context:** Reviewing effectiveness of automated learning extraction vs agent-based approach.
**Learning:** The regex-based learning extraction script was only catching ~3% of actual learnings due to overly strict pattern matching. Agent judgment found 10 learnings where script found 2. Automated pattern matching can appear to be working while being largely ineffective.
**Stakes:** high (knowledge retention system fundamentally broken, years of lost learning)
