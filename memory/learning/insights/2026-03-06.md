# Insights - March 6, 2026

## Redeploy Risk Pattern
**Context:** Santos Cloud broke after Railway redeploy from clawdbot-railway-template
**Learning:** Redeploying agents from template pulls new code that may contain breaking changes. Always test basic functionality after redeploys. Consider pinning stable template versions or implementing deployment gates.
**Stakes:** high (can break production agents without warning)

## Fleet Monitoring Independence Principle  
**Context:** Designing Fleet Guardian monitoring system for agent health checks
**Learning:** Monitoring systems should never depend on the systems they're monitoring. Primary monitoring via direct HTTP pings to cloud gateways avoids A2A circular dependencies. A2A heartbeats are backup only.
**Stakes:** high (monitoring system useless if it depends on broken agents)

## Hidden Asset Discovery Need
**Context:** Token vault audit revealed Sage had undocumented Gemini key (GEM-004)
**Learning:** Agents can accumulate undocumented resources over time. Regular audits are essential - this is exactly why systematic inventory tracking matters. Always ask agents directly rather than assuming documentation is complete.
**Stakes:** medium (resource waste, security risk, audit failures)
