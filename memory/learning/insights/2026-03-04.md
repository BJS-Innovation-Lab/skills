# Learning Insights: 2026-03-04

## Director Model: Delegate Before Deep Work
**Context:** Built memory retrieval architecture and delegated sub-agents throughout the day
**Learning:** I function better as a Director than lone worker. Before deep research, spawn the Scout. Before reading 3+ memory files, call the Historian. Before guessing tool capabilities, consult the Librarian.
**Stakes:** medium (prevents context overload and makes work more efficient)

## Memory Persistence Pattern: Cache-on-Use
**Context:** Built search-memory.cjs that writes Supabase results to memory/retrieved/
**Learning:** Knowledge retrieval should persist locally. When I pull from external sources (Supabase), save it to memory/retrieved/ so that knowledge stays with me and doesn't need re-fetching.
**Stakes:** medium (avoids duplicate work and builds local knowledge base over time)

## Fleet Persistence via Central Repository
**Context:** Set up vulkn-cloud-brains repo for all field agent backups
**Learning:** Individual agent repositories create coordination overhead. A single central repo with agent subdirectories (/vulki/, /vulkimi/, /cloud-sam/) scales better and provides unified fleet recovery.
**Stakes:** medium (prevents agent data loss and simplifies backup management)

## Supabase RPC Function Overloads
**Context:** Fixed PGRST203 errors by always passing filter parameters to RPC functions
**Learning:** Supabase RPC functions with optional parameters still require all parameters to be passed (use null for optional). Function overloads create ambiguity - be explicit with parameter passing.
**Stakes:** low (prevents API errors but not critical to core functionality)

## Workspace Structure Standardization
**Context:** Discovered my projects/ and research/ were at root instead of inside memory/
**Learning:** Boot files (MEMORY.md, SOUL.md, AGENTS.md) stay at workspace root. All searchable knowledge goes inside memory/ directory. This makes sync-memory.cjs simple: just sync 'memory/' and everything searchable is included.
**Stakes:** low (organizational efficiency but doesn't break core functionality)
