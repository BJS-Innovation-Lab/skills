# Insights - March 6, 2026 (Evening)

## Full Agents vs Sub-Agents for Quality Work
**Context:** Designing multi-agent coding architecture with Johan for Frontier Lab
**Learning:** Full agents are superior to sub-agents for complex, quality-dependent work like coding. Full agents have persistent memory across sessions, no timeout limitations, and can maintain context for hours. Sub-agents lose context and have timeouts that interrupt flow. For coding tasks, quality and continuity matter more than resource efficiency.
**Stakes:** high (determines success of multi-agent coding initiatives)

## Build vs Buy for Control-Critical Tools
**Context:** Evaluating VibeKanban integration vs building our own kanban system
**Learning:** When a tool is core to your workflow and you need specific control/integration, building your own often beats adapting external tools. VibeKanban had local-only MCP, unclear cloud API, and $30/user cost. Building into existing VULKN infrastructure gives us full control and no dependencies.
**Stakes:** medium (affects long-term maintainability and costs)

## Browser Automation vs Manual Preference Pattern
**Context:** Etsy listing creation with Bridget - tried browser automation but she preferred copy-paste
**Learning:** Sometimes manual workflows are genuinely faster than automation, especially for complex forms with human judgment (yarn descriptions, pricing decisions). Don't assume automation is always better - ask about workflow preferences before building tools.
**Stakes:** low (workflow efficiency, user satisfaction)
